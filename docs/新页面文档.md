下面是一份落地的“新增页面一致化”清单 + 可直接复用的页面脚手架，按它做就能让新页面与现有页面保持同一套设计与交互风格。

一、页面一致性清单

- 结构与容器
  - 使用 PageLayout 作为页面根容器。
  - 页面主体使用 page-rounded-container；内部再放 content-wrapper。
  - 顶部可选 page-header（主标题/副标题）。
  - 加入背景装饰 background-decoration（三个 circle + float 动画）。
- 样式与主题
  - 顶部引入 ../../styles/common.scss，统一变量与阴影/圆角：var(--text-primary)、var(--border-light)、var(--shadow-light)、var(--radius-\*)。
  - 主色使用 #7F4515，按钮沿用 primary-btn/secondary-btn 渐变与线框风格。
  - 统一卡片组件：ModernCard；卡内小标题、分组、间距与其他页面一致。
- 状态与交互
  - 统一三种状态：LoadingScreen、EmptyState、内容区。
  - 网络异常与业务错误使用 uni.showToast；401 统一清 Token 并跳转登录页。
  - 有外链用与 Dashboard 相同的外链打开逻辑（H5 window.open / App plus.runtime.openURL / 其他端复制链接）。
- 登录校验与数据请求
  - onLoad/onShow 统一校验 token；无 token 弹 Toast 并 reLaunch 到 /pages/index/index。
  - 请求统一从 getApp().globalData.apiBaseURL 取 baseURL，并在 header 携带 Authorization: Bearer <token>。
- 适配与动画
  - 保持与现有页面一致的 padding、grid/flex 间距、icon 尺寸（uni-icons 16/20/24/30）。
  - 背景 circle + float 动画、卡片 hover/active 轻动效。
- 工程与入口
  - 在 pages.json 注册新页面，配置 navigationBarTitleText。
  - 如需从“主页-核心功能”进入，在 index.vue 的 features 中新增入口。

二、可复用的页面脚手架
直接新建页面并粘贴下面代码，然后按需修改标题、接口与内容区块。

          </ModernCard>

  </PageLayout>

<script setup>
import { ref, computed } from "vue";
import { onLoad, onShow } from "@dcloudio/uni-app";
import PageLayout from "../../components/PageLayout/PageLayout.vue";
import ModernCard from "../../components/ModernCard/ModernCard.vue";
import LoadingScreen from "../../components/LoadingScreen/LoadingScreen.vue";
import EmptyState from "../../components/EmptyState/EmptyState.vue";

const isLoading = ref(true);
const data = ref(null);

const hasData = computed(() => !!data.value);

onLoad(() => {
  if (!ensureLogin()) return;
  uni.setNavigationBarTitle({ title: "新功能页面标题" });
  fetchData();
});

onShow(() => {
  if (!ensureLogin()) return;
});

function ensureLogin() {
  const token = uni.getStorageSync("token");
  if (!token) {
    uni.showToast({ title: "请先登录", icon: "none" });
    uni.reLaunch({ url: "/pages/index/index" });
    return false;
  }
  return true;
}

async function fetchData() {
  isLoading.value = true;
  const token = uni.getStorageSync("token");
  const baseURL = getApp().globalData.apiBaseURL;

  try {
    const { statusCode, data: res } = await uni.request({
      url: `${baseURL}/api/v1/your-endpoint`,
      method: "GET",
      header: { Authorization: "Bearer " + token },
    });

    if (statusCode === 200 && (res.success || res.code === 200)) {
      data.value = res.data || res.result || {};
    } else if (statusCode === 401) {
      uni.removeStorageSync("token");
      uni.showToast({ title: "登录已过期，请重新登录", icon: "none" });
      setTimeout(() => uni.reLaunch({ url: "/pages/index/index" }), 1500);
    } else {
      uni.showToast({
        title: res.detail || res.message || "获取数据失败",
        icon: "none",
      });
      data.value = null;
    }
  } catch (e) {
    console.error("请求失败", e);
    uni.showToast({ title: "网络连接失败", icon: "none" });
    data.value = null;
  } finally {
    isLoading.value = false;
  }
}

function handlePrimary() {
  uni.showToast({ title: "已执行主操作", icon: "success" });
}

function handleSecondary() {
  fetchData();
}
</script>

三、把页面挂到导航与入口

- 在 pages.json 添加：
  - path: pages/new-feature/index
  - style.navigationBarTitleText: 新功能页面标题
- 在“主页-核心功能”（/pages/dashboard/index.vue）features 中新增：
  - { text, description, icon, url: "/pages/new-feature/index" }

四、接口与状态的统一细节

- 请求头统一加 Authorization: Bearer <token>；401 统一清除 token、toast 提示、延迟 reLaunch 登录页。
- 错误信息优先展示后端返回的 detail/message；未覆盖时显示“网络连接失败”或“获取数据失败”。
- Loading/Empty/Content 三态互斥，避免闪烁。

按以上清单与脚手架落地，新页面就能无缝融入现有设计系统。
